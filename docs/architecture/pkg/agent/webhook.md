## Overview

Implements the outbound push pipeline that batches all enriched events from every eBPF monitor and POSTs them as a JSON array to a configurable HTTP endpoint. The intended consumer is a ClickHouse ingest listener.

This file also defines the `WebhookEvent` wire format and all supporting flat structs designed for direct ClickHouse column insertion, so the ingest program receives pre-computed analytics signals without additional processing.

### `EventMetadata`

Pre-computed analytics carrier. Contains everything ClickHouse needs for fast queries without joining the sparse context fields.

| Field | JSON key | Description |
|---|---|---|
| `EventID` | `event_id` | UUID v4, generated per event |
| `Source` | `source` | Always `"ebpf"` |
| `Severity` | `severity` | Highest violation severity, or event's own severity, or `"INFO"` |
| `RiskScore` | `risk_score` | Computed: CRITICAL×25 + HIGH×20 + MEDIUM×10 + LOW×5, capped at 100 |
| `ComplianceStatus` | `compliance_status` | `"violation"` or `"clean"` |
| `ViolationCount` | `violation_count` | Length of violations slice |
| `CISControls` | `cis_controls` | De-duplicated list of control IDs |
| `SeverityDistribution` | `severity_distribution` | Count per severity bucket |
| `ThreatIndicators` | `threat_indicators` | String tags (see below) |
| `Workload` | `workload` | `WorkloadMeta` — flat K8s identity |
| `SecurityPosture` | `security_posture` | `SecurityPosture` — flat security flags |
| `EventDetail` | `event_detail` | `EventDetail` — flat type-specific fields |

### `ThreatIndicators` string values

Generated by `buildThreatIndicators()`. De-duplicated before returning.

| Tag | Trigger |
|---|---|
| `root_process` | `Process.UID == 0` or `File.UID == 0` or `Capability.UID == 0` |
| `privileged_container` | `Container.Privileged == true` |
| `allow_privilege_escalation` | `Container.AllowPrivilegeEscalation == true` |
| `host_namespace_escape` | `Container.HostNetwork || HostPID || HostIPC` |
| `missing_resource_limits` | Both memory and CPU limits are empty |
| `no_seccomp` | `SeccompProfile == ""` or `"Unconfined"` |
| `no_apparmor` | `ApparmorProfile == ""` |
| `unsigned_image` | `Container.ImageSigned == false` |
| `unscanned_image` | `ImageScanStatus != "" && != "scanned"` |
| `excess_rbac_permissions` | `Kubernetes.RBACLevel >= 2` |
| `stale_sa_token` | `ServiceAccountTokenAge > 7,776,000 s` (90 days) |
| `no_network_policy` | `!Kubernetes.HasDefaultDenyNetworkPolicy` |
| `automount_sa_token` | `Kubernetes.AutomountServiceAccountToken == true` |
| `sensitive_file_access` | `File.Sensitive == true` |
| `dangerous_capability` | `Capability.Name` in dangerous set |

**Dangerous capabilities:** `CAP_SYS_ADMIN`, `CAP_NET_ADMIN`, `CAP_NET_RAW`, `CAP_SYS_PTRACE`, `CAP_SYS_MODULE`, `CAP_DAC_OVERRIDE`, `CAP_SETUID`, `CAP_SETGID`

## `WebhookPusher`

### Struct fields

| Field | Type | Description |
|---|---|---|
| `config` | `WebhookConfig` | Resolved config (defaults applied in constructor) |
| `clusterID` | string | Stamped on every `WebhookEvent` |
| `nodeName` | string | Stamped on every `WebhookEvent` |
| `client` | `*http.Client` | Configured with TLS transport and timeout |
| `eventCh` | `chan WebhookEvent` | Buffered channel, capacity = `BatchSize × 4` |
| `logger` | `*zap.Logger` | Structured logger |
| `done` | `chan struct{}` | Closed by `Stop()` |
| `wg` | `sync.WaitGroup` | Tracks `flushLoop` goroutine for drain guarantee |
| `startOnce` | `sync.Once` | Prevents double-goroutine on repeated `Start()` calls |
| `stopOnce` | `sync.Once` | Prevents double-close of `done` channel |

### `NewWebhookPusher(cfg, clusterID, nodeName, logger) *WebhookPusher`

Guards against zero/negative values that would panic at runtime:
- `BatchSize ≤ 0` → 100
- `FlushInterval ≤ 0` → 5 s (prevents `time.NewTicker(0)` panic)
- `Timeout ≤ 0` → 10 s (prevents infinite HTTP block)

Builds the `http.Client` with a TLS-configured transport via `buildTLSTransport()`.

### `Start(ctx context.Context)`

Launches `flushLoop` in a goroutine. Protected by `startOnce` — safe to call multiple times.

### `Stop()`

1. Closes `done` (via `stopOnce` — idempotent)
2. `wg.Wait()` — blocks until `flushLoop` finishes its drain+flush

Without the `wg.Wait()`, a fast `Stop()` could return before the final batch POST completes.

### `Send(event *enrichment.EnrichedEvent, violations []*rules.Violation)`

Constructs a `WebhookEvent` via `buildWebhookEvent()` and non-blocking-sends it to `eventCh`. Logs a warning and drops the event if the channel is full (back-pressure relief valve).

### `flushLoop(ctx context.Context)`

Background goroutine. Accumulates events into a local slice.

**Flush triggers:**
- Batch reaches `BatchSize` → immediate flush
- Ticker fires (`FlushInterval`) → flush whatever has accumulated
- `done` closed or ctx cancelled → `drainAndFlush()` then return

### `drainAndFlush(batch *[]WebhookEvent)`

Drains any remaining events from `eventCh` into `batch` (non-blocking), then POSTs using a fresh `context.Background()` with the configured timeout. Uses a fresh context because the agent context is already cancelled during SIGTERM shutdown.

Receives a pointer to `batch` so capacity-growth appends are not lost by value semantics.

### `post(ctx context.Context, batch []WebhookEvent) error`

JSON-marshals the batch and POSTs to `config.TargetURL`.

**Behaviours:**
- Sets `Content-Type: application/json`
- Sets custom headers from `config.Headers`
- Retries once on transient network errors (timeout, connection refused, connection reset, EOF) — not on 4xx/5xx
- Drains the response body before `Close()` to allow TCP connection reuse
- On `status >= 400`, logs `redactHeaders(config.Headers)` (not original values)

### `buildTLSTransport(cfg WebhookConfig) http.RoundTripper`

Builds an `http.Transport` with a `*tls.Config`:
- `TLSCAPath != ""` → appends custom CA PEM to system pool
- `TLSCertPath != "" && TLSKeyPath != ""` → loads client certificate for mTLS
- Zero values → system CA pool, no client cert (safe default)

### `redactHeaders(headers map[string]string) map[string]string`

Returns a copy with values for `authorization`, `x-api-key`, and `x-auth-token` replaced with `"[redacted]"`. Only used on the error log path — the actual HTTP request always receives original values.

## Event Construction Helpers

### `buildWebhookEvent(event, violations, clusterID, nodeName) WebhookEvent`

Assembles the full `WebhookEvent` from an `EnrichedEvent`. Copies all context pointers by reference (no deep clone needed — events are immutable after enrichment). Calls `buildMetadata()`.

### `eventTypeName(raw string) EventType`

Normalises raw event type strings to canonical constants. Handles both short forms (`"process"`) and long forms (`"process_execution"`). Falls through to `EventType(raw)` for unknown types.

### `buildMetadata(event, violations, summaries) EventMetadata`

Computes all analytics signals:
- Violation counts and de-duplicated CIS control IDs
- `SeverityDistribution` counters
- `RiskScore` (CRITICAL=25, HIGH=20, MEDIUM=10, LOW=5, capped at 100)
- Highest severity label
- `WorkloadMeta` via `buildWorkload()`
- `SecurityPosture` via `buildSecurityPosture()`
- `ThreatIndicators` via `buildThreatIndicators()`
- `EventDetail` via `buildEventDetail()`

### `newEventID() string`

Generates a UUID v4 string using `crypto/rand`. Format: `xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx`.

### `dedupStrings(in []string) []string`

Order-preserving deduplication. Used to collapse repeated `"root_process"` indicators.
